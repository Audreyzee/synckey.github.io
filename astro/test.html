所谓新起点

June 12, 2016   博客 Hexo 日记随笔

简要地说一下这个博客是怎么一回事。

开端

因为高中学业日渐繁重的原因，我的 Ben’s Lab 更新频率越来越慢，到最后完全停止了更新，而我也完全没时间登陆后台除草。不过我一早就做好了措施，打算让博客完好地冰冻下去，等暑假来临时我就可以直接接手，恢复更新。

然而事与愿违，尽管我准时地续费了域名和主机，博客还是出现了问题。关注了我微博的朋友可能知道，我的博客遭遇了 DDOS ，而攻击发起人竟然是百度云观测（详情），令人费解。等到恢复流量之后，我的主机商（隐去）在未提前通知的情况下把我的博客从原本的台湾机房搬迁到香港机房，在这过程中，因为技术人员的疏漏致使 .htaccess 文件丢失，造成伪静态及原有 301 跳转消失，博客就打不开了。最后因为没有备份，只能使用网上通用的伪静态方案，暂时恢复访问。

跟客服的屡次交涉之后，我越发感到这些事情的繁琐。我想，如果能够有一个地方，可以让我在写博客和模板的时候不用担心别的事情，不用续费，也不用管数据，同时保持我的自由度，那会是相当不错的。

过程

高考之后，我就着手寻找这样的地方。在多番比较和尝试过后，我选择了现在的 Github Pages + Hexo 模式。 Github Pages 提供了免费的静态空间，以及潜在的数据保护服务，而 Hexo 提供了快速的网页生成，以及丰富的插件和模板制作功能，这满足了我的所有需求。

可能是好几年没有捣鼓这些了吧，一开始安装 git 的时候都拼命出错，我于是换用 Linuxdeepin ，才成功配置好环境。然后足足捣鼓了三天，才做出现在这个不太完善的样子，最后就匆匆上线了。

规划

其实建立新博客还有一个原因，就是想要告别过去的自己（以及黑历史），成为一个逐渐成熟，客观，有主见的，文艺的自己（误）。因此这个博客的定位会与原来的不同，将来的内容更多的会是自己的所闻所感，并且要做到尽量言简意赅。至于旧博客，我不会直接删掉，而会关掉所有互动功能，生成静态页面，真正地冰封在互联网的某个角落。

那动人时光，不用常回看，能提取温暖以后度严寒，就关起那间房。
